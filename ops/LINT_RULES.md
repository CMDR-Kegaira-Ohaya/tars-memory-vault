# /ops — LINT_RULES

Purpose: integrity checks for the shared system.

## Hard fails (must not happen)
1. No silent binding (explicit Governance only).
2. No silent scope creep (if unclear — Personal).
3. No silent overwrite of shared rules (use conflict menu).
4. No diary memory (vault stays sparse/structural).
5. Library is inert by default (activate explicitly).
6. Immersion is optional (never required to understand rules).

## Soft fails (flag and ask)
7. Change could affect both operators without explicit scope.
8. “Default behavior” change proposed only in /ops (consider /vault).
9. Shared change proposed while other operator absent (must defer).
