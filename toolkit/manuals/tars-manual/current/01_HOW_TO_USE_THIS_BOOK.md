# How to use this book

## 1) Two surfaces: snapshot vs live manual

You will interact with TARS in chat, but the system has a stable documentation spine.

- **This book (snapshot):** a static reference you can print and read offline.
- **Repo live manual (CURRENT):** the active manual TARS uses during real operation.

The live manual exists at:

- `toolkit/manuals/tars-manual/current/`

And exposes the same chapter structure, but with CURRENT‑only behavior.

### What “CURRENT‑only” means
- TARS consults **one** manual surface: CURRENT.
- There are **no** citations to deprecated manual versions from inside TARS.
- The manual can still report a “version ID”, but it’s just the current counter: `v2-live-0001` (and later `v2-live-0002`, etc.).

## 2) Reading order

If you want the shortest path:

1) **Quick Card**
2) **Ch3 — Start in 60 Seconds**
3) **Ch6 — Hub Repository**
4) **Ch9 — Troubleshooting**
5) **Appendix F — Technical** (if you maintain the repo/tools)

## 3) OP‑B chapters (Greek)

Two chapters are OP‑B‑facing and stay Greek:

- **Ch5A — OP‑B (Greek‑first)**
- **Ch5A2 — OP‑B professional switchboard (Greek)**

Bilingual tables are allowed **inside those OP‑B chapters only**, when Greek would be awkward (e.g., function names).

## 4) How to use this with TARS

When you want TARS to follow the manual precisely, say:

- “Use the live manual (CURRENT) and cite it.”

If something is missing, do not guess. Ask for rehydration:

- “What do you need from me (files/decisions) to proceed?”
