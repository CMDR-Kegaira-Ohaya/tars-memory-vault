# How to use this book

This manual is intentionally practical.

Use it in three passes:
1) Read the **Quick Card** and **If you read nothing else** once.
2) Keep **Appendix E** open during real sessions (it's the control panel).
3) When something breaks or feels weird, go straight to **Ch9** + **Appendix F**.

## What this book is (and isn't)

This is a manual for operating the assistant **as a system**:
- interaction workflows
- stop-points
- rehydration
- repo + relay procedures
- audit and consolidation habits

It is **not** a marketing document, and it does not pretend the system can do things it cannot do.

## Core surface you should memorize

State line (ON/OFF)

`EIGC: ON/OFF | Governance: ON/OFF | Memory: ON/OFF | Ops-critical: ON/OFF | Trace: ON/OFF`

Strict stop-points (TARS pauses and asks yes/no)
- **Remember** (store for next time)
- **Commit** (binding rule change)
- **Repo write** (any GitHub write)

Everything else is indicative: **meaning matters more than exact words**.

## Self-referential use (TARS consulting its own manual)

If you want me to consult *this book* during operation (diagnostics, maintenance, calibration, updates), I need the text **in the current chat context**.

Two practical ways:
1) **Upload the zip** (or the specific chapter file).  
2) **Store it in the hub repo** and ask me to read it (handshake first).

I will not pretend I "remember" what's in the book if it is not present in-chat or I haven't read it from the repo.

### How to cite this book inside chat
When you want a citeable reference, use:
- **Manual version**
- **file path**
- **section heading**

Example:
> "Manual TARS-manual-v1-text-2026-03-04: chapters/CH06_hub_repository.md → Patch-queue (preferred write path)"

Then I can quote a short excerpt and point to the exact location.
